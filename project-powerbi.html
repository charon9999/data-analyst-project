Project Overview

A 4-page interactive Power BI dashboard built on the Retail Sales Dataset (62,884 transactions across 8 product categories, 58 stores, and 9 countries from 2016–2021). The dashboard provides executive-level KPIs, sales trend analysis with moving averages, product-level profitability rankings, and customer demographic segmentation — enabling stakeholders to make data-driven decisions at every level.

Development Process

End-to-end workflow from raw Excel data to a polished interactive dashboard:

1

Data Load & Import

Loaded 62,884 rows from Retail Sales Dataset.xlsx into Power BI using Power Query. Verified column types, handled null values, and validated data integrity.

2

Data Cleaning (Power Query)

Promoted headers, set correct data types (dates, decimals, integers). Filtered out anomalies. Split and trimmed text columns for consistency.

3

Data Modeling (Star Schema)

Built a star schema with fact and dimension tables. Established one-to-many relationships between orders, customers, stores, products, and categories.

4

Date Table Creation

Created a dedicated Date dimension table using DAX CALENDARAUTO() with Year, Quarter, Month, MonthName, and WeekDay columns for time intelligence.

5

DAX Measures

Authored calculated measures: Total Revenue, Total Profit, Profit Margin %, AOV, Total Units Sold, Customer Count, MoM Growth, and Moving Averages.

6

Calculated Columns

Added computed fields: Line Total (Qty × Price), Profit (Revenue − Cost), Profit %, Customer Age, and Age Group binning for demographic analysis.

7

Visual Design & Layout

Designed 4 focused pages with KPI cards, line charts, bar charts, donut charts, maps, tables, and area charts. Applied consistent color theme and formatting.

8

Interactivity & Filters

Added slicers for Year, Country, Category, and Customer. Configured cross-filtering between visuals and drill-through capabilities for detail exploration.

Date Table & Time Intelligence

A dedicated Date dimension table is essential for time intelligence functions in Power BI. Created using DAX for seamless integration with the data model:

DAX
// Create Date Table
DateTable =
VAR MinDate = MIN(Orders[OrderDate])
VAR MaxDate = MAX(Orders[OrderDate])
RETURN
    ADDCOLUMNS(
        CALENDAR(MinDate, MaxDate),
        "Year",        YEAR([Date]),
        "Quarter",     "Q" & QUARTER([Date]),
        "Month",       MONTH([Date]),
        "MonthName",   FORMAT([Date], "MMMM"),
        "MonthShort",  FORMAT([Date], "MMM"),
        "WeekDay",     FORMAT([Date], "dddd"),
        "DayOfWeek",   WEEKDAY([Date], 2)
    )

The Date table is marked as a date table in the model and linked to the OrderDate column in the fact table, enabling DAX time intelligence functions like DATEADD, SAMEPERIODLASTYEAR, and TOTALYTD.

DAX Measures

Custom DAX measures powering all KPI cards, charts, and dynamic calculations across the dashboard:

DAX
// Total Revenue
Total Revenue = SUMX(Sales, Sales[Quantity] * RELATED(Products[Price]))

// Total Cost
Total Cost = SUMX(Sales, Sales[Quantity] * RELATED(Products[Cost]))

// Total Profit
Total Profit = [Total Revenue] - [Total Cost]

// Profit Margin %
Profit Margin % = DIVIDE([Total Profit], [Total Revenue], 0) * 100

// Total Orders
Total Orders = DISTINCTCOUNT(Sales[OrderNumber])

// Total Units Sold
Total Units Sold = SUM(Sales[Quantity])

// Average Order Value
Avg Order Value =
    DIVIDE(
        [Total Revenue],
        [Total Orders],
        0
    )

// Avg Customer Value
Avg Customer Value =
    DIVIDE(
        [Total Revenue],
        DISTINCTCOUNT(Sales[CustomerID]),
        0
    )

// Monthly Avg Revenue
Monthly Avg Revenue =
    AVERAGEX(
        VALUES(DateTable[MonthName]),
        [Total Revenue]
    )

// Revenue Month-over-Month Growth %
MoM Growth % =
    VAR CurrentMonth = [Total Revenue]
    VAR PreviousMonth =
        CALCULATE(
            [Total Revenue],
            DATEADD(DateTable[Date], -1, MONTH)
        )
    RETURN
        DIVIDE(CurrentMonth - PreviousMonth, PreviousMonth, 0) * 100

// City Moving Average (for Sales Analysis page)
City Moving Avg =
    AVERAGEX(
        DATESINPERIOD(DateTable[Date], MAX(DateTable[Date]), -3, MONTH),
        [Total Revenue]
    )

Data Model

Star schema design with a central fact table (Sales/Order Line Items) surrounded by dimension tables for optimal query performance and clean relationships:

Fact Table

  • Sales (Order Line Items) — 62,884 rows: TransactionID, OrderNumber, LineItem, ProductID, Quantity, OrderDate, DeliveryDate, CustomerID, StoreID

Dimension Tables

  • Customers — 11,887 records: CustomerID, Name, Gender, City, State, Country, Continent, DOB
  • Products — 2,492 records: ProductID, Name, Brand, Color, Cost, Price, SubcategoryID
  • Product Categories — 8 categories with 32 subcategories (hierarchy)
  • Stores — 58 stores: StoreID, Country, State, SqMeters, OpenDate
  • DateTable — Auto-generated calendar: Date, Year, Quarter, Month, MonthName, WeekDay

Relationships

  • Sales[CustomerID] → Customers[CustomerID] (Many-to-One)
  • Sales[ProductID] → Products[ProductID] (Many-to-One)
  • Sales[StoreID] → Stores[StoreID] (Many-to-One)
  • Sales[OrderDate] → DateTable[Date] (Many-to-One)
  • Products[SubcategoryID] → Subcategories[SubcategoryID] (Many-to-One)
  • Subcategories[CategoryID] → Categories[CategoryID] (Many-to-One)

Calculated Columns

DAX calculated columns added to enrich the data model for analysis:

DAX
// Line Total (Revenue per line item)
Line Total = Sales[Quantity] * RELATED(Products[Price])

// Line Cost
Line Cost = Sales[Quantity] * RELATED(Products[Cost])

// Line Profit
Line Profit = Sales[Line Total] - Sales[Line Cost]

// Profit % per line
Profit % = DIVIDE(Sales[Line Profit], Sales[Line Total], 0) * 100

// Customer Age
Customer Age =
    DATEDIFF(
        Customers[DOB],
        TODAY(),
        YEAR
    )

// Age Group (for demographic analysis)
Age Group =
    SWITCH(
        TRUE(),
        Customers[Customer Age] < 25, "18-24",
        Customers[Customer Age] < 35, "25-34",
        Customers[Customer Age] < 45, "35-44",
        Customers[Customer Age] < 55, "45-54",
        Customers[Customer Age] < 65, "55-64",
        "65+"
    )
