DAX Measures

Custom DAX measures powering the dashboard KPIs:

DAX
// Total Revenue
Total Revenue = SUM(orders[total_amount])

// Total Orders
Total Orders = COUNTROWS(orders)

// Average Order Value
Avg Order Value = DIVIDE([Total Revenue], [Total Orders], 0)

// Return Rate
Return Rate =
    DIVIDE(
        COUNTROWS(returns),
        COUNTROWS(orders),
        0
    ) * 100

// Revenue Month-over-Month Growth
MoM Growth =
    VAR CurrentMonth = [Total Revenue]
    VAR PreviousMonth =
        CALCULATE(
            [Total Revenue],
            DATEADD(orders[order_date], -1, MONTH)
        )
    RETURN
        DIVIDE(CurrentMonth - PreviousMonth, PreviousMonth, 0) * 100

// Customer Count
Unique Customers = DISTINCTCOUNT(orders[customer_id])

// Avg Delivery Days
Avg Delivery Days = AVERAGE(deliveries[days_to_delivery])

// Discount Impact
Total Discount = SUM(orders[discount_amount])

Data Model

The Power BI data model mirrors the normalized MySQL schema with star-schema relationships:

Fact Table

  • orders — Central fact table with revenue metrics, linked to all dimensions

Dimension Tables

  • customers — Customer ID, type (New/Returning)
  • products — SKU, name, category, unit price
  • locations — City and state for geographic analysis
  • discounts — Discount codes and percentages
  • order_items — Line-level detail with quantity and pricing
  • deliveries — Warehouse, delivery date, transit time
  • returns — Return reasons, refund status, return window
